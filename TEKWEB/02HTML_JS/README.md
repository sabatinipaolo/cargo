### 02HTML_JS
galleria di immagini fatta con  HTML + javascript ( va be' a parte un po' di bootstrap)


pro: 

- gli **skill aumentano**: HTML + JAVASCRIPT, bisogna saper programmare ! ( va be' anche un po' di bootstrap )

- l'app è costituita da una **singola pagina**

- se modifico il contenuto o l'aspetto (style) devo modificare solo la **singola pagina** 

- non c'è nessuna elaborazione lato server, il tempo di startup dipende solo dal download. Questo apparentemente trascurabile non lo è se si considera che l'app potrebbe essere ospitata su server condivisi e sovraccarichi

- Una volta caricata la pagina, si ricaricano solo le immagini richieste senza scaricare altro HTML

contro:

- statico: se aggiungo una immagine, devo modificare il codice della pagina.
