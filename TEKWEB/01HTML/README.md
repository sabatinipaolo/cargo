### 01HTML
galleria di immagini fatta con puro e semplice HTML ( va be' a parte un po' di bootstrap)


pro: 

- gli skill richiesti sono basilari: solo HTML ( va be' anche un po' di bootstrap ), nessuna competenza di programmazione è richiesta

- non c'è nessuna elaborazione lato server, il tempo di startup dipende solo dal download. Questo apparentemente trascurabile non lo è se si considera che l'app potrebbe essere ospitata su server condivisi e sovraccarichi

- per cambiare immagine si scarica un intera pagina HTML, ma grazie alla cache questo impatta solo la prima volta.


contro:

- statico: se aggiungo una immagine, devo aggiungere un file e modificarne 2 !

- statico: se modifico il contenuto o l'aspetto (style) devo modificare tutti i file
