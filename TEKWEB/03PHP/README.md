### 03PHP

galleria di immagini fatta con  HTML + PHP ( va be' anche un po' di bootstrap)

pro:
- gli **skill aumentano**: HTML + PHP, bisogna saper programmare ! ( va be' anche un po' di bootstrap )

- l'app è costituita da una **singola pagina**

- se modifico il contenuto o l'aspetto (style) devo modificare solo la singola pagina 

- **dinamico** : Basta aggiungere un immagine nella cartella e l'app la rileva automaticamente

- Una volta caricata la pagina, si ricaricano solo le immagini richieste senza scaricare altro HTML

contro:
- Quando scorro le immagini di fatto viene richiesta una nuova pagina (un po' come con l'HTML puro), in più:

- elaborazione lato server, il tempo di risposta dipende dal tempo di escuzione del PHP sul server. Questo apparentemente trascurabile non lo è se si considera che l'app potrebbe essere ospitata su server condivisi e sovraccarichi
